🇮🇳 DPDPA 2023

India's Digital Personal Data Protection Act — Framework Overview

About DPDPA 2023
India's landmark data privacy law enacted in August 2023, establishing comprehensive obligations for data fiduciaries and data processors.
JurisdictionIndia
EnactedAugust 2023
Key Controls12 Controls
Max PenaltyUp to ₹250 Crore
Key Requirements
Core compliance obligations under DPDPA 2023
Consent ManagementCritical
Data Fiduciary ObligationsCritical
Data Principal RightsMajor
Breach Notification (72h)Major
Cross-Border TransfersStandard
Children's Data ProtectionCritical
Compliance Tips
Recommended actions for DPDPA readiness
• Appoint a Data Protection Officer (DPO)
• Implement double opt-in consent management
• Create and publish a clear privacy notice
• Set up 72-hour breach notification procedures
• Implement parental consent for children's data
• Review all third-party data processor agreements

🌐 ISO 27001

International Standard for Information Security Management Systems

About ISO 27001:2022
The globally recognized standard for information security management systems (ISMS), covering 93 controls across 4 themes.
Issued ByISO / IEC
VersionISO 27001:2022
Total Controls93 Controls (Annex A)
CertificationAvailable
Key Control Domains
ISO 27001 four control themes
Organizational Controls (A.5)Critical
People Controls (A.6)Major
Physical Controls (A.7)Major
Technological Controls (A.8)Critical
Compliance Tips
ISO 27001 readiness checklist
• Define the ISMS scope and security objectives
• Conduct a formal risk assessment and treatment
• Implement relevant Annex A controls
• Conduct internal audits and management reviews
• Perform annual policy reviews
• Pursue certification with an accredited body

🇪🇺 GDPR

EU General Data Protection Regulation

About GDPR
The EU's comprehensive data protection regulation effective since May 2018, applying to all organizations that process EU residents' personal data.
JurisdictionEuropean Union + EEA
EffectiveMay 25, 2018
Articles99 Articles
Max Fine€20M or 4% Global Revenue
Key Requirements
GDPR core compliance obligations
Lawful Basis for ProcessingCritical
Data Subject RightsCritical
Privacy by DesignMajor
DPO AppointmentMajor
72-Hour Breach NotificationCritical
Compliance Tips
GDPR readiness steps
• Map all personal data flows across the organization
• Establish a lawful basis for each processing activity
• Implement data subject request procedures
• Conduct DPIAs for high-risk processing activities
• Review all data processor agreements (Art. 28)
• Appoint a Data Protection Officer where required

The security standard for all organizations that handle, store, or transmit cardholder data, maintained by the PCI Security Standards Council.
Issued ByPCI SSC
Versionv4.0 (2022)
Requirements12 Core Requirements
Applies ToAll Card Processors
Key Requirements
PCI DSS 12 requirements summary
Network Security (Req 1–2)Critical
Cardholder Data Protection (Req 3–4)Critical
Vulnerability Management (Req 5–6)Major
Access Control (Req 7–9)Major
Monitoring & Testing (Req 10–11)Standard
Compliance Tips
PCI DSS readiness steps
• Define and protect the cardholder data environment
• Encrypt all transmission of cardholder data (TLS 1.3)
• Implement MFA for all CDE access
• Deploy IDS/IPS systems
• Conduct quarterly vulnerability scans
• Perform annual penetration testing

📋 NIST CSF 2.0

NIST Cybersecurity Framework 2.0 — Govern, Identify, Protect, Detect, Respond, Recover

About NIST CSF 2.0
The NIST Cybersecurity Framework 2.0, published in 2024, provides guidance for managing and reducing cybersecurity risk across all sectors and organization sizes.
Issued ByNIST (USA)
Version2.0 (February 2024)
Functions6 (GV, ID, PR, DE, RS, RC)
ApplicabilityAll sectors, all sizes
Key Functions
NIST CSF 2.0 six core functions
Govern (GV) — NEW in 2.0Critical
Identify (ID)Critical
Protect (PR)Critical
Detect (DE)Major
Respond (RS)Major
Recover (RC)Standard
Compliance Tips
NIST CSF 2.0 readiness steps
• Define organizational risk governance and risk tolerance
• Maintain a complete asset inventory with classification
• Conduct formal risk assessments at least annually
• Implement identity, access, and data protection controls
• Deploy continuous monitoring and anomaly detection
• Document and test incident response and recovery plans
• Address supply chain and third-party cybersecurity risks
